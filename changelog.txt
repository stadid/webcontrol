Webcontrol by Stadid
Complete easy to use self-hosted standalone solution built on top of ebusd to replace Vaillant remote control system (MultiMATIC, SensoAPP, MyVaillant, etc.)

Version 2024.08.15 (IMAGE)
=========================
* Fully refactored ALL program code to support licensing mechanism (split into a BASIC and FULL version)
BASIC version limits to one zone/circuit and one user-defined parameter.

+ Added display of relative air humidity for the controller in VRC700 (for VRC700-assigned Zone only)

+ Added support for the VRC720 family controllers:
+ VRC720, VRC720/2, VRC720/3

Added support for ecoTEC plus heaters:
art. 0010024664, model VU 20CS/1-5 (N-INT4)
art. 0010024665, model VU 25CS/1-5 (N-INT4)
art. 0010024666, model VU 30CS/1-5 (N-INT4)
art. 0010024667, model VU 35CS/1-5 (N-INT4)
art. 0010024668, model VUW 26CS/1-5 (N-INT4)
art. 0010024669, model VUW 32CS/1-5 (N-INT4)
art. 0010024670, model VUW 36CS/1-5 (N-INT4)

* Updated VRC700 code and the built-in EBUSD configuration to support full zones/circuits for the corresponding controllers generation:
+ VRC700 (first generation) – 2 zones/circuits
+ VRC700/2 – 3 zones/circuits
+ VRC700/4 – 3 zones/circuits
+ VRC700/5 – 9 zones/circuits
+ VRC700/6 – 9 zones/circuits

+ Added Telegram bot commands "VPN" and "VPN ON".
Bot command "VPN" shows VPN status; "VPN ON" enables the VPN connection. Useful if VPN was manually disabled and access to the device is lost

+ Added Telegram bot command "address" (or "addresses")
Shows all IP addresses (Ethernet, Wi-Fi, VPN) assigned to the device (mini-PC with Webcontrol)

+ Developer Tools section now includes the option to set an interval of the periodic actions. This parameter defines the frequency of, for example, system error checks, Telegram bot command processing, Narodmon reporting, etc.
The previously it was set to 10 mins. In the current version it can be set to at least 5 mins.

+ Added EBUS adapter connection status and bus status to the diagnostics section.

Version 2025.02.01 (IMAGE + CODE)
=========================
+ Added an option to display room temperatures and humidities measured by the controller on the "Status" page (despite a zone assignment of the controller)
This option is necessary for systems where controller is not assigned to any zone.

+ Added setting to make the "Parameters" page default. When enabled, accessing the program opens the "Parameters" page instead of "Status."

+ Added program update via Internet.

* The set of displayed heat-generator parameters on the "Parameters" page is extended. In addition to the generator status code, temperatures of supply/return lines are shown (depends on heat generator model and firmware version).

+ Added support for heat generators in cascade — now the following parameters are displayed (status code, supply/return lines temperatures, as well as error code if fault). If a generator faults, a notification is sent.

+ Added Telegram bot command "report".
The "report" command executes the "status" and "parameters" commands and returns a merged output from both.

* Improved time/date correction procedure for the controller — time/date are no longer corrected at 00:00 and 12:00 to avoid affecting cascade operation during the switch.

[!] WARNING! This version is SETTINGS-COMPATIBLE (db file) with all previous versions EXCEPT 2023.08.08.
